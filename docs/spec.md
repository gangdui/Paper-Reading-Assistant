# 软件需求规格说明

## 功能需求

### F1：论文发现

用户可以在“发现论文”页面输入研究方向或关键词，例如“AIGC 模型水印”。系统应在用户点击按钮后才触发检索，避免页面刷新时重复请求外部 API。

### F2：关键词扩展

系统使用 DeepSeek API 将中文研究方向扩展为英文关键词和多个英文检索式。DeepSeek 只用于关键词扩展和推荐理由辅助，不允许直接生成论文列表。

### F3：真实论文检索

系统优先使用 Semantic Scholar API 检索论文。若未配置 `SEMANTIC_SCHOLAR_API_KEY`，系统应尝试公开检索。若 Semantic Scholar 请求失败或受到限流，系统切换到 arXiv 预印本数据源。若真实检索均失败，系统使用 mock fallback 示例数据。

论文字段应尽量包含：

- title
- authors
- year
- venue
- abstract
- citationCount
- url
- openAccessPdf
- paperId

### F4：文献核查 Agent

系统对每篇论文进行基础核查：

- 来源是否可追溯。
- 标题、作者、年份、链接等元数据是否完整。
- 是否有 PDF 链接。
- venue 或来源层面的初步质量判断。

核查结果可包括：

- 来源核查：可追溯 / 待确认。
- 元数据：完整 / 部分完整。
- 质量判断：顶会顶刊 / 正式发表 / arXiv 预印本 / 待进一步确认。

系统必须说明：质量判断不等于论文结论可靠，只表示来源或 venue 层面的初步判断。

### F5：论文推荐与阅读建议

系统展示 Top 3 优先关注论文和完整论文推荐列表。每篇论文展示：

- 标题
- 年份 / 数据源 / venue / 引用数
- PDF 状态
- 推荐优先级：高 / 中 / 低
- 为什么推荐
- 阅读建议
- 核查摘要
- 论文链接和 PDF 链接

英文 abstract、作者列表、缺失字段、命中关键词、推荐分数和 paperId 默认折叠。

推荐理由和阅读建议只能基于 API 返回的 title、abstract、year、venue、citationCount、source 和关键词命中情况生成，不允许编造作者、年份、会议、标题或链接。

### F6：PDF 文本提取与清洗

用户可以上传单篇 PDF。系统使用 `pdfplumber` 提取文本，并进行基础清洗：

- 去除多余空行和连续空格。
- 修复英文断词，例如 `meth-\nod` 合并为 `method`。
- 合并不必要的换行。
- 尽量去除页码、重复页眉页脚和水印。
- 保留论文结构词，如 Abstract、Introduction、Method、Experiment、Conclusion、References。

### F7：结构化文献笔记生成

系统将清洗后的论文文本发送给 DeepSeek API，最多发送前 15000 个字符。AI 输出中文结构化文献笔记，包含 9 个模块：

1. 一句话总结
2. 论文摘要
3. 研究背景与研究问题
4. 主要内容概述
5. 提出的方法
6. 实验设计与结果分析
7. 主要贡献
8. 不足与局限
9. 研究生阅读建议

如果原文没有明确说明某部分内容，应输出“原文未明确说明”。

### F8：Markdown 下载

生成的结构化文献笔记可以以 Markdown 形式下载，便于课程报告、文献整理和后续复盘。

### F9：错误处理与 fallback

系统应处理以下情况：

- DeepSeek API Key 缺失。
- DeepSeek 网络错误、余额不足或模型名错误。
- Semantic Scholar 未配置 Key 或公共限流。
- arXiv 请求失败。
- PDF 无法解析或文本为空。
- 课堂 demo 时外部 API 不可用。

错误提示应友好，不应导致页面崩溃。fallback 数据必须明确标注，不应伪装成真实推荐。

## 非功能需求

- 简单性：代码结构清晰，适合课程 MVP 讲解。
- 可运行性：能够通过 `streamlit run app.py` 启动。
- 稳定性：API 失败不导致页面崩溃。
- 可验证性：主要流程可以通过测试用例和手动验收清单检查。
- 可信边界：不让 AI 编造论文列表；真实论文元数据必须来自 API。
- 可扩展性：后续可扩展 OpenAlex、Crossref、DBLP、OCR 和文献库保存。
- 安全性：真实 API Key 放在 `.env`，不提交到仓库。

## 输入输出

### 输入

- 研究方向或关键词。
- 用户上传的单篇 PDF。
- `.env` 中的 `DEEPSEEK_API_KEY`。
- 可选的 `SEMANTIC_SCHOLAR_API_KEY`。

### 输出

- 英文关键词和检索式。
- 真实 API 返回的论文列表。
- Top 3 优先关注论文。
- 每篇论文的推荐理由、阅读建议和核查摘要。
- 中文结构化文献笔记。
- 可下载的 Markdown 文件。
- 错误提示和 fallback 状态说明。

## 边界条件

- Semantic Scholar 未配置 Key：使用公开检索，可能受限。
- Semantic Scholar 受限或失败：切换到 arXiv 预印本。
- arXiv 失败：使用 mock fallback 示例数据。
- arXiv 论文：不能直接判断为高质量论文，正式发表信息需要后续核查。
- mock fallback：仅用于 demo，不代表真实推荐。
- 扫描版 PDF：可能无法提取文本，当前 MVP 暂不支持 OCR。
- 论文过长：仅发送清洗后前 15000 个字符给模型。
- API Key 缺失：页面显示友好提示。

## 验收标准

- 用户可以输入研究方向并触发论文发现。
- 系统展示 Top 3 优先关注论文和完整论文列表。
- 论文标题、链接、年份、venue 等信息来自真实 API 或明确的 mock 示例。
- arXiv 结果标注为预印本，并提醒正式发表信息需要后续核查。
- 文献核查摘要显示来源、元数据和质量判断。
- 英文 abstract 默认折叠。
- 用户可以上传 PDF 并生成结构化文献笔记。
- 用户可以下载 Markdown 笔记。
- API Key 缺失、限流或网络错误时，页面不崩溃并显示友好提示。

## 不做的内容

当前 MVP 暂不实现：

- 多篇 PDF 批量上传。
- 用户登录和历史记录。
- OCR 扫描版 PDF 识别。
- 引文网络分析。
- 自动生成正式参考文献库。
- 对论文结论可靠性作最终判断。
- 生产级权限、并发和部署能力。
