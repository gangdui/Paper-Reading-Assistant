# 项目想法

## 项目名称

面向研究生的科研论文发现与阅读助手

## 项目定位

本项目从早期的“PDF 论文阅读与结构化文献笔记生成助手”升级为“科研论文发现与阅读助手”。它不仅帮助用户阅读已经下载好的 PDF 论文，也帮助用户从研究方向出发发现真实论文、核查论文来源，并筛选值得进一步阅读的候选文献。

当前项目面向课程 MVP 和后续产品原型展示，重点关注研究生科研阅读中的最小闭环：

```text
发现论文 -> 核查论文 -> 打开 PDF -> 生成结构化阅读笔记
```

## 目标用户

- 需要快速进入某个研究方向的研究生。
- 正在准备课程论文、开题报告、文献综述或组会汇报的学生。
- 需要从大量论文中筛选候选文献的科研初学者。
- 希望把论文阅读结果整理成结构化笔记的课程项目小组。

## 使用场景

1. 用户输入研究方向，例如“AIGC 模型水印”，系统扩展英文关键词并检索候选论文。
2. 用户查看 Top 3 优先关注论文，快速判断哪些论文值得先读。
3. 用户查看每篇论文的来源、venue、PDF 状态、推荐理由、阅读建议和核查摘要。
4. 用户打开感兴趣论文的 PDF，切换到“阅读 PDF”页面。
5. 用户上传 PDF，生成中文结构化文献笔记并下载 Markdown。

## 核心痛点

- 不知道应该用哪些英文关键词检索论文。
- 搜索结果很多，但难以判断哪些论文值得先读。
- AI 容易编造论文标题、作者、会议和链接，需要真实 API 约束。
- arXiv 预印本和正式发表论文容易被混淆。
- 论文篇幅长、术语多，初读成本高。
- PDF 文本提取常出现换行混乱、断词、页眉页脚等噪声。
- 文献笔记格式不统一，后续整理和复盘效率低。
- 课堂 demo 可能遇到 API Key、网络或限流问题，需要 fallback 保证稳定性。

## MVP 目标

本项目的 MVP 目标是完成一个可运行、可验证、适合课堂展示的科研论文助手：

1. 支持研究方向输入和英文关键词扩展。
2. 使用真实论文 API 检索论文，不让 AI 编造论文列表。
3. 支持 Semantic Scholar、arXiv 和 mock fallback。
4. 对论文来源、元数据完整度和质量判断进行基础核查。
5. 展示 Top 3 优先关注论文和完整论文列表。
6. 为每篇论文提供推荐理由和阅读建议。
7. 支持上传 PDF、文本提取、文本清洗和结构化文献笔记生成。
8. 支持 Markdown 下载和错误提示。

## 项目价值

- 对研究生：降低找论文和初读论文的门槛，提高文献调研效率。
- 对课程项目：体现从需求、设计、实现、测试到文档的 SDD 流程。
- 对 AI 辅助编程：展示如何用 AI 协作完成 MVP 迭代、调试、UI 产品化和文档更新。
- 对产品原型：形成一个可演示的科研助手流程，但不夸大系统能力，不声称能保证论文质量。

## 重要边界

- 系统不能保证论文质量，只能基于来源、venue 和元数据做初步判断。
- arXiv 结果是预印本，正式发表信息需要后续核查。
- mock fallback 只是 demo 示例数据，不代表真实推荐。
- 结构化笔记用于辅助阅读，不能替代用户精读原文。
